· Postmark · drop-in component ·

One file, every location.

Save postmark.js to your site (anywhere — /public/postmark.js in Astro works), then use it any of the four ways below. Type in the inputs to see the live update.

Live — bound to inputs

// Whenever your location changes:
Postmark.render(document.getElementById("stamp"), {
  city:  currentStop.city,    // "MOAB"
  state: currentStop.state,   // "UT"
  date:  currentStop.date,    // "OCT 26"
  year:  currentStop.year,    // "25"
});

Or pure HTML — auto-initialized

Any element with a data-postmark attribute gets rendered on page load. Server-render the attribute values from your trip data — no JS to write.

BRYCE · UT
MONTROSE · CO
JACKSON · WY
CAPITOL REEF · UT · 2025
ALBUQUERQUE · NM
<div data-postmark
     data-city="MOAB"
     data-state="UT"
     data-date="OCT 26"
     data-year="25"
     data-rotate="-6"></div>

// In Astro you'd just template it from your stop data:
<div data-postmark
     data-city={stop.city.toUpperCase()}
     data-state={stop.state}
     data-date={formatStamp(stop.arrival)}
     data-year={String(stop.year).slice(-2)}/>

Get the SVG string

For server-side rendering, OG images, email, etc. — pass options, get back a self-contained SVG string. Inject it as inline HTML or write it to a file.

const svg = Postmark.svg({ city: "MOAB", state: "UT", date: "OCT 26", year: "25" });
container.innerHTML = svg;

Use as an <img> src

Handy when you want a draggable, downloadable, easily-styled image (set transform: rotate, drop shadows, etc).

img.src = Postmark.dataUrl({ city: "MOAB", state: "UT", date: "OCT 26", year: "25" });

Looks great on dark too

// For dark backgrounds, pass the brighter red:
Postmark.render(el, { city, state, date, year, color: "#E85844" });

That's it. postmark.js is the only file you need — drop it in /public/ and load it once.
